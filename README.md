# Top Anime Project

## Live Demo

[View the live site](https://josesiva-code.github.io/anime-evaluation-project/)

## Project Description

Top Anime Project is a dynamic web page built with HTML, CSS and JavaScript.

The project uses the Jikan API to get a list of top anime and then displays only the anime with a score higher than 9. The page is created dynamically with JavaScript, which means the main content is generated by the script instead of being written directly in the HTML.

The visual style was inspired by Japanese/anime design, with a samurai-themed header, custom katana cursors, anime-style typography and a warm rising-sun background.

## Project Goal

The main goal of this project is to practice JavaScript DOM manipulation, API requests and dynamic page creation.

This project shows how JavaScript can:

- Create HTML elements dynamically.
- Fetch data from an external API.
- Filter API results.
- Display data on the page.
- React to user actions without refreshing the page.
- Update the interface when the user clicks on an anime image.

## Main Features

- Dynamic page structure created with JavaScript.
- External CSS file for all styling.
- Anime data fetched from the Jikan API.
- Anime list filtered by score higher than 9.
- Responsive anime card grid.
- Anime image and title displayed in each card.
- Side details panel shown when an anime image is clicked.
- Details panel includes score, rating, release date, genres, type, episodes, status and synopsis.
- Sticky details panel that stays visible while scrolling.
- Highlight effect on the selected anime card.
- Dynamic footer with the current year.
- Samurai image beside the main title.

- Red katana cursor when hovering over anime images.
- Japanese-inspired visual theme.

## API Used

This project uses the Jikan API, an unofficial API for MyAnimeList data.

Endpoint used:

```text
https://api.jikan.moe/v4/top/anime
```

The API returns information about top anime, including title, image, score, rating, release date, genres and synopsis.

## How The Project Works

1. JavaScript creates the main application container.
2. JavaScript creates the header, main content area, anime grid, details panel and footer.
3. The project fetches anime data from the Jikan API.
4. The API response returns a list of top anime.
5. The code filters the list and keeps only anime with `score > 9`.
6. Each filtered anime is displayed as a card.
7. Each card contains an anime image and title.
8. When the user clicks an anime image, the side panel appears.
9. The side panel is filled with more information about the selected anime.
10. The selected card receives a visual highlight.

## Score Filter

The project only displays anime with:

```js
score > 9
```

This means the page only shows anime with a very high score according to the data returned by the Jikan API.

## File Responsibilities

### `index.html`

Contains the basic HTML document structure and connects the CSS and JavaScript files.

### `style.css`

Controls the full visual style of the project, including:

- Layout
- Anime grid
- Details panel
- Typography
- Background
- Samurai image
- Card styles
- Custom cursors
- Responsive visual structure

### `script.js`

Controls the dynamic behavior of the project, including:

- Creating page elements
- Fetching data from the API
- Filtering anime by score
- Creating anime cards
- Handling image clicks
- Showing anime details
- Updating the selected card
- Creating the footer year automatically

### `assets/`

Stores the visual assets used by the project, such as the samurai image and custom katana cursor images.
